📋 프로젝트 개요 ▼

이 프로젝트는 웹 개발과 이미지 처리에 필요한 다양한 도구들을 모은 컬렉션입니다.

🎯 주요 목표

  • 이미지 처리 작업의 자동화
  • 반복적인 웹 개발 작업의 효율화
  • 사용하기 쉬운 인터페이스 제공
  • 외부 서버 없이 브라우저에서 완전한 처리

✨ 특징

  • Client-side 처리: 모든 작업이 브라우저에서 처리되어 보안성 확보
  • 반응형 디자인: 모바일과 데스크톱에서 모두 최적화
  • 모던한 UI: 블랙앤화이트 미니멀 디자인
  • 고품질 처리: Canvas API를 활용한 고품질 이미지 처리

🎨 팔레트 유형

1. 🎨 대표 컬러 (Dominant Colors)

K-means 클러스터링을 통한 주요 색상 추출. 브랜딩, 주요 UI 컬러로 활용.

2. 🌈 보색 팔레트 (Complementary Colors)

색상환에서 180° 정반대 색상들. 강렬한 대비와 액센트 컬러로 활용.

3. 🌅 배경용 팔레트 (Background Colors)

높은 명도와 낮은 채도로 가독성 최적화. 웹/앱 배경으로 활용.

4. 🌿 아날로그 대비 (Analogous Contrast)

색상환에서 인접한 색상들의 조화로운 대비. UI 액센트로 활용.

5. 🌡️ 온도 대비 (Temperature Contrast)

따뜻함과 차가움의 감정적 대비. CTA 버튼, 강한 시각적 포인트로 활용.

📸 썸네일용 (다중 업로드)

  • 최대 7개 이미지 드래그 앤 드롭 업로드

  • 610×477 사이즈로 변환
  • 총 7개 파일 생성

📄 본문용 (단일 업로드)

  • 1개 이미지 업로드
  • 정방형 12사이즈: 60, 120, 125, 130, 160, 180, 210, 240, 303, 470, 500, 800px
  • 2가지 형식: PNG, JPG
  • 총 24개 파일 생성 (파일명: 60.png, 120.jpg 등)

🔧 스마트 크롭 알고리즘

원본 이미지 크기를 고려한 2단계 처리:

  1. 원본 보호: 1200px 이상이면 먼저 축소
  2. 타겟 고려: 타겟 크기를 채울 수 있는 최소 크기로 확대
  3. 중앙 크롭: 여분 부분은 중앙 기준으로 크롭
  4. 여백 방지: 타겟을 완전히 채우도록 보장

예시: 1920×1080 → 800×800 변환

  • 1단계: 1200×675로 축소 (원본 보호)
  • 2단계: 세로 기준으로 1422×800 크기로 확대
  • 3단계: 가로 311px씩 크롭 → 800×800 완성
  • 결과: 상품 전체가 보이면서 여백 없음!
